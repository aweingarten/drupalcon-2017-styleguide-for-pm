WTF is styleguide driven development?

Start with visual designs.

Break it apart into visually modular elements

Build a small model with drupal agnostic markup and styling.

Here is a design

Here is that design exploded into modular elements

Lets take 1 element of the design and develop it.

Step 1: forget drupal. The horrible traumatic markup, etc does not exist.

This is an optimistic methodology.

Step 2: What content will this thing have.

We will first start with a JSON that describes what is going to be dropped into template

Step 3: What does the html for the thing look like.

We'll build a html example using the variables we described in JSON.

We'll put it some pretty CSS classes that we can work with.

Step 4: Style the above by writing a small sass file.

Step 5: Mix it up and get a styleguide!

Step 6: Integrate!

Import the code into drupal, get drupal to print markup that is consistent with your model.

Step 6: Enjoy!

Now we have a styled widget in drupal!.

And it's powered by your styleguide.

Step 7: Your client threw you a change request!

Go and make your change to the styleguide.

Recompile all the things.

Step 8: Enjoy at your own risk!
